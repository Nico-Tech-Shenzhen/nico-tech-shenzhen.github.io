# Search External Activity Audit

Date: 2026-06-20

## Current search implementation

- `/search/` uses PaperMod's fast search script at `assets/js/fastsearch.js`.
- The script loads `../index.json`, so `/search/` resolves the search index from site root `/index.json`.
- Site JSON output is enabled by `config.toml` with `home = ["HTML", "RSS", "JSON"]`.
- The root JSON index is generated by `layouts/_default/layouts_defaultindex.json.json`.
- Fuse.js defaults to searching `title`, `permalink`, `summary`, and `content`.
- The result renderer requires non-empty `title` and `permalink`, then renders each result as a clickable `.post-entry`.

## Notes

- Existing internal Hugo page search records are still generated first and use the previous title, permalink, section, summary, and content fields.
- External records set `permalink` to `source_url`, so search results are clickable without creating Hugo pages.
- Searchable external text includes title, summary, source label, source platform, activity type, and language.
- The blank-result fix remains in place: the template filters empty titles and permalinks, and the search script also drops empty-title or empty-permalink records before building the Fuse index.
- Search result rendering now shows `[source_label]` for data-backed external records and escapes rendered title, permalink, and aria-label text.
- Hugo was unavailable locally (`hugo` command not found), so verification used static template review, data audits, and JavaScript syntax checking.
